## 使用策略

做项目研究时的标准工作流：

1. **新项目初筛** → RootData（融资+团队） + DropsTab（综合） + CoinGecko（市值/价格）
2. **基本面深挖** → DeFiLlama（TVL/收入/费用） + Token Terminal（财务报表）
3. **链上行为分析** → Nansen / Arkham（聪明钱和大户） + Dune（自定义查询）
4. **风险排查** → Tokenomist（解锁抛压） + Bubblemaps（筹码集中度）
5. **空投机会** → CryptoRank Drophunting + Alpha Drops
6. **叙事/新闻** → The Block（英文原文） + Messari（研究报告）
7. **Dune 数据面板** → 使用 `templates/dune-token-dashboard.sql` 模板，替换合约地址和链名即可生成代币交易数据面板（日交易量/交易次数/活跃钱包/新增地址/买卖压力）
8. **找智能合约地址** → SPA dApp 不像传统网站，合约地址嵌在 JS 里。先分析 JS bundle（`references/contract-discovery-spa.md`），再查链上持币地址反推，最后才用 web search

## 空投/链游项目快速评估方法论

Ian 问过调研方法，标准四步：

1. **信息收集**（并行）— Twitter/X 看官方说什么 → RootData 查融资/团队 → CryptoRank Drophunting 查活动时间线 → 多源交叉验证（至少 3-5 个源）
2. **关键变量提取** — 有没有代币？TGE 时间？锁仓规则？入场成本？奖池规模？
3. **成本收益判断** — 最关键一步。零成本 + TGE 快 + 即刻解锁 = 高优先级。高成本 + TGE 远 + 长锁仓 = 低优先级
4. **策略输出** — 不是「值不值得撸」，而是「怎么撸最优」：注册时机、推荐杠杆、卖出策略

本质：**信息筛选 → 交叉验证 → 关键变量提取 → 成本收益判断**。不靠单一信源，不信任何「必撸」标签。

## Dune 数据面板快速生成

当 Ian 需要为某个代币创建 Dune 链上数据面板时：

1. 先用 CoinGecko API 查合约地址和链
2. 参考 `references/dune-dashboard-patterns.md`（含日度统计/Counter/买卖压力 3 种模式），替换合约地址和链名
3. 交付 SQL 代码给 Ian（优先直接粘贴，而非下载文件），附带 Dune 建图→组 Dashboard 的步骤

Dune 表结构注意：`dex.trades` 是跨链聚合表，字段含 `blockchain` / `token_bought_address` / `token_sold_address` / `amount_usd` / `taker` / `block_time`。

## Twitter 数据归档分析

收到 Twitter 数据导出包（.zip）时：
1. 解压到 `/tmp/twitter_data/`
2. 运行 `scripts/twitter_archive_stats.py /tmp/twitter_data/data/`
3. 输出：账号概况、互动数据、内容结构（原创/回复比例）、高互动推文、发帖时段分布
4. 基于输出给 Ian 诊断建议（内容策略调整方向）

当 Ian 需要找预测市场（Polymarket/Kalshi/MEXC）的足球/体育投放 KOL 时，参考 `references/prediction-market-kol-research.md`。核心要点：

- **fxtwitter API 验证粉丝数** → `curl -s https://api.fxtwitter.com/HANDLE`
- **MEXC 无 badge 体系** → 走 Affiliate Program（referral link），联系 @LeoInCrypto

## 九、Web3 / TG 社区工具定价研究

当 Ian 要给 Sentinel 或 Web3/TG/DC 社区工具定价时，优先参考 `references/web3-community-bot-pricing-benchmarks.md`。里面包含 Combot、Rose、GroupHelp、Varta、Watchdog、Collab.Land、Guild、Collony、Phantom Mod 的公开 usage / pricing / paid-user 推断方法。
