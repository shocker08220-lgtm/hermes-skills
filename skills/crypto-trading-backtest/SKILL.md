# Crypto 交易策略回测

当 Ian 提出一个交易策略想验证时，用本框架跑回测。**核心原则：先诊断再回测，不等完整结果才知道问题。**

## 环境

```
Python: ~/.pyenv/versions/3.12.3/bin/python3
依赖: ccxt, pandas, numpy
安装: ~/.pyenv/versions/3.12.3/bin/pip3 install ccxt pandas numpy
数据源: Binance (ccxt.binance())
```

## 方法论：三步回测流程

### Step 1 — 诊断条件命中率（必做，不能跳过）

直接跑完整回测 = 浪费时间。先检查每个条件的独立命中率，详见 `references/diagnostic-methodology.md`。

```
# 对每个周期单独统计：
# - MACD 金叉状态占比（macd_line > signal）
# - MACD 零轴上方占比
# - 两者同时满足占比
# - Bollinger 做多信号占比
#
# 然后在 5m 粒度上检查多周期共振率
```

如果某个条件命中率 < 3%，说明那是瓶颈，需要调优。

### Step 2 — 定位瓶颈并调优

常见瓶颈：
- **12H MACD 零轴上方**：BTC 12H MACD 慢线很少翻到零轴以上，和 1H/4H 共振概率接近 0%。**必须降级为"只要求金叉状态"或"不管零轴"。**
- **三重完全共振**：三个周期同时满足太难。改用 2/3 或加权。
- **Bollinger 叠加**：中轨斜率 + 价格在轨位再加一层，信号密度骤降。

### Step 3 — 跑回测 + 看出场原因分布

跑完必须看 `exit_reason` 分布。如果 macd_rev 吃掉 100% 的出场 → TP/SL 是摆设，需要调参数。

## 关键 Pitfalls

### Pitfall 1: 12H MACD 零轴要求扼杀所有信号
```
症状：60 天 0 笔交易
原因：12H MACD 慢线（EMA14）翻到零轴需要巨大趋势动能
     震荡/回调行情永不上零轴，三周期共振概率 = 0%
修复：12H 只要求 macd_line > signal（金叉状态），不管零轴
```

### Pitfall 2: MACD 反转出场统治 TP/SL
```
症状：所有出场都是 macd_rev，TP(4%)/SL(2%) 从不触发
原因：1H MACD 反转比 2-4% 价格移动来得快
     等于入场的同周期条件触发出场，自相矛盾
修复方向：
  A) 去掉 macd_rev，纯 TP/SL
  B) 收紧 TP 到 1.5%、SL 到 1%（匹配 1.84 盈亏比）
  C) 退出条件用更大周期（4H MACD 反转而非 1H）
```

### Pitfall 3: 做空在牛市里天然劣势
```
BTC 偏多行情里做空胜率 44% vs 做多 67%
修复：加 BTC > EMA200 过滤器，仅在此条件下允许做空
     或策略直接只做多
```

### Pitfall 4: 15m MACD 出场噪音过大
```
症状：胜率 27.9%，最大连亏 8 笔，均持仓仅 1.5h
      所有的盈利趋势都被 15m 死叉的小回调震出去
      BTC 在上升趋势中频繁 15m 死叉，每根都砍仓
根因：15m MACD 对日内噪音过于敏感
      趋势还在但 15m 级别微回调就触发死叉
修复：出场改用 1H MACD 反转（而非 15m）
      胜率从 27.9% → 31.2%，持仓从 1.5h → 5.0h
      累计收益从 +1.41% → +3.34%
```

### Pitfall 5: "方向消失就做空" 是毒药（v7 血泪教训）
```
症状：v7 回测 BTC 60天，胜率 25%，累计 -11.27%
      17/36 笔被止损（47%），做空 10 笔胜率仅 10%
根因：
  ① 方向=None 时硬做空 → BTC 震荡/回调不是反转，在震荡区做空送钱
  ② 15m 死叉+空头排列出场太慢 → 价格跌穿 1% 止损线条件才满足
  ③ 17 次止损平均亏 -1.24%，把做多 15m_dead 赚的 +11.73% 全部吃掉
修复：
  - 方向=None 不交易（不猜方向，宁可错过不可做错）
  - 出场用 1H MACD 反转，不用 15m（见 Pitfall 4）
  - Ian 原话："改回上个版本" — 测试失败立刻回滚到已知最优版
```

### Pitfall 6: 六条件 AND 门扼杀方向稳定性
```
症状：大方向只占 11% 的时间，87% 出场是 dir_change
      六个条件（1H/4H/12H MACD金叉+零轴 + Boll方向）
      断裂分布均匀：各占 14-18%，没有单一瓶颈
根因：六个 AND 条件的概率乘法效应
      不是某个指标差，是"全都要"导致系统脆弱
修复方向：
  A) 降维到 1H MACD 金叉+零轴做主方向
  B) 用 EMA 排列代替 MACD 共振
  C) 接受 11% 的时间窗口，专注在那个窗口反复做波段
```

### Pitfall 6: CCXT 请求慢，需要耐心
```
ccxt 导入本身就慢（5-10秒），给 timeout >= 180
数据拉取多周期时用 background=true + notify_on_complete
```

## 策略架构模式：v6 最优版（已跨币种验证）

经过 7 轮迭代后稳定的策略结构。**BTC/ETH/SOL 60 天回测累计 +33.38%，163 笔交易，胜率 42.9%，盈亏比 2.29。**

```
1. 大方向判断（每根 15m bar 检查一次）
   1H MACD(7,14) 金叉状态 + 零轴上方  ← 主方向
   4H MACD 金叉状态（不要求零轴）     ← 确认
   1H Bollinger 中轨向上              ← 趋势健康度
   → 方向 = "long" 或 "short" 或 None
   ⚠️ 12H 已砍掉 — 对日内策略太慢且不增加价值

2. 方向锁定后只做一个方向
   方向 = long → 只做多
   方向 = short → 只做空
   方向 = None → 不交易（不等、不猜）
   方向改变 → 强制平仓

3. 入场：15m MACD 金叉事件 + MA5>MA10>MA20 + 放量(≥1.5x前5均量)

4. 出场：1H MACD 反转 或 止损 -1%
   不能用 15m 出场 — 见 Pitfall 4

5. 方向不变可重新入场
   出场后等下一个 15m 金叉 → 再入场
```

### 跨币种验证结果（60天，Binance）

| 币种 | 交易数 | 胜率 | 累计收益 |
|---|---|---|---|
| BTC | 54 | 37% | +7.67% |
| ETH | 55 | 44% | +14.19% |
| SOL | 54 | 48% | +11.52% |
| **汇总** | **163** | **42.9%** | **+33.38%** |

关键设计决策：
- **1H 主 + 4H 确认（砍 12H）**：12H 零轴要求让三周期共振概率归零，砍掉后信号从 32 笔 → 54 笔，累计翻倍
- **15m 进 + 1H 出**：入场用快周期抓时机，出场用慢周期拿趋势
- **Bollinger 只要求中轨方向**：价格跌回下轨无所谓，Ian 明确说"只要通道往上走就可以"
- **不做空或慎做空**：做多 115 笔贡献 +24.58%，做空 48 笔贡献 +8.80%
- **方向=None 不交易**：不要猜方向，宁可错过不可做错。v7 证明"方向消失就做空"是毒药

## 迭代优化模式

当用户要求"改 X 为 Y 并重跑"时：
1. **只改相关函数**（通常是 `big_direction()` 或出场逻辑），其余流水线保持不变以公平对比
2. **打印对比表**：与上一版并排显示胜率、交易数、累计收益、盈亏比
3. **始终保存结果 CSV** 供后续参考
4. **版本命名递增**（v6 → v7），失败版回滚到上一已知最优版本
5. **诊断优先于完整回测** — 先用 Step 1 诊断条件命中率，不要盲目跑完整回测

## 指标计算规范

Ian 策略的参数：
```
MACD: 快线=7, 慢线=14, 信号线=9
MA: 5, 10, 20
Bollinger: 周期=20, 标准差=2
成交量确认: 当前量 ≥ 前5根均量的 1.5 倍
```

MACD 金叉"状态"（macd_line > signal）与金叉"事件"（上一根不满足、这一根满足）是两个概念。回测大方向判断用状态，入场用事件。
